# Lab2: Binary Semantic Segmentation

## 作業目標

在 Oxford-IIIT Pet 資料集上實作二元語義分割，將每張圖片的每個 pixel 分類為**前景（寵物本體）**或**背景**，並以 Dice Score 作為評估指標上傳 Kaggle 競賽。

---

## 資料集

- **來源**：Oxford-IIIT Pet Dataset
- **標註格式**：Trimap PNG，pixel 值定義如下：
  - `1` → 前景（寵物本體）→ binary mask `1.0`
  - `2` → 背景 → binary mask `0.0`
  - `3` → 邊界（不確定區域）→ binary mask `0.0`
- **Split**：使用 Kaggle 提供的指定 split 檔案（`train.txt` / `val.txt` / `test_*.txt`）
- **輸入大小**：統一 resize 至 256×256，以 ImageNet mean/std 正規化

---

## 實作要求

### Model 1：UNet（嚴格遵照 2015 年原始論文）

**必須遵守：**
- 禁止使用 BatchNorm
- 所有 Conv2d 禁止 padding（`padding=0`）
- 上採樣只能使用 `ConvTranspose2d`（2×2），不可用 bilinear
- Skip connection 必須使用 center crop，不可用 padding 對齊

**允許的調整：**
- `in_channels`：1 → 3（RGB 輸入）
- `out_channels`：2 → 1（single channel sigmoid，做 binary segmentation）

**架構（256×256 輸入）：**
```
Encoder:  64 → 128 → 256 → 512 → 1024 ch（含 Bottleneck）
Decoder:  1024 → 512 → 256 → 128 → 64 ch
Output:   Conv1×1 + sigmoid
```

---

### Model 2：ResNet34 + UNet Decoder

**架構：**
```
Encoder（ResNet-34）:
  stem   → 64ch,  128×128
  layer1 → 64ch,   64×64  （3 × BasicBlock）
  layer2 → 128ch,  32×32  （4 × BasicBlock）
  layer3 → 256ch,  16×16  （6 × BasicBlock）
  layer4 → 512ch,   8×8   （3 × BasicBlock）

Decoder（UNet-style）:
  dec4: 512 → 256  (skip: layer3)
  dec3: 256 → 128  (skip: layer2)
  dec2: 128 →  64  (skip: layer1)
  dec1:  64 →  64  (skip: stem)
  dec0:  64 →  32  (no skip)
  out:   32 →   1  (Conv1×1 + sigmoid)
```

---

## 訓練設定

| 項目 | 設定 |
|------|------|
| Optimizer | Adam，`weight_decay=1e-5` |
| Scheduler | CosineAnnealingLR，`eta_min=1e-6` |
| Loss | `0.5 × BCE + 0.5 × Dice Loss` |
| Checkpoint | 只在 val Dice 更新最佳值時儲存 |

---

## 評估方式

在驗證集上計算 **Dice Score**，以 `threshold=0.5` 將輸出機率圖二值化後計算，對每個 batch 平均後再整體平均。

---

## Inference 輸出格式

輸出 Kaggle 格式的 CSV，欄位為 `image_id` 與 `encoded_mask`。

**注意**：mask 必須先以 PIL NEAREST 插值 resize 回各圖片的**原始尺寸**，再進行 column-major RLE 編碼，否則 Kaggle 解碼後 mask 會完全錯位。
